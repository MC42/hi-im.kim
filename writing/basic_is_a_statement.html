The Web Is Opinionated

Regardless of how much someone might want to use sites written in gobs and heaps of Javascript transpiled webassembly, you inevitably will. This is not an accident, this is an intentional choice.

A designer, a programmer, a vibe programmer decided that, yes, for money or time, we should use a toolkit that is predecated on someone else's fight not interfering with using said tool.

Is this what the end user might want? That doesn't truly matter, not with a dark pattern that continuously disrespects what their intents are. After all, sites are emphemeral and both can and will vanish regardless of the user wants.

We're in for an eternity of consent dialogues that misunderstand consent, countless clicks of [ Not Now ] that might never go away.

Maybe if I'm not so cynical though, I'd point out that in small circles, the web ring never died.

Human-centric design still an intentional act of creation turned into obligation by every byte transferred. Maybe we can mourn websites that were beloved in their time and might never come back, CSS hacks spewed into Cohost's void with joyous intent.

It's okay to make bad websites, it's wonderful to make silly websites! And sometimes, maybe sometimes, the right move is still to make something the same way you did when you were younger.

Maybe re-enacting your first fight with vim will bring back that childish whimsy of a web that was bold and fresh, optimistic and open, huh? Sorry, I'll try to dial back the cynicism.

Or maybe it's time to run rm -rf / --no-preserve-root and wipe the slate clean. Sheep farming seems ideal, right?
