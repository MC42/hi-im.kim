Mangling Subtitles For Fun and No Profit

I spend some time trying to better explain workflow(s) for using subtitles in video editing.

What are subtitles anyways?

Subtitles are the wonderful, fantastic tool that allows text to be shown in a video or livestream to explain what's going on. While they don't always, they can indicate the speaker, emphasis on certain words, pauses, and most importantly, markup.

The "Technology Connections" channel very specifcally makes use of subtitles on YouTube.

Historically, formats like the .ASS file format only included very limited markup capability, only including a few XHTML tags that enabled colors, bold, italics, and underline. Some modifications even enabled the use of intentional positioning, which is really cool, but that was then and this is now.

Modern subtitle containers allow you to do some pretty wild stuff, beyond just positioning and location. During a viewing of Shoujo Kagegi Revue Starlight, the use of both Japanese and English characters co-mingle on the screen. Some of the text is video, some of it is subtitles; can you guess which is which?

to swap images and test your guess!

With the presumption of a cynical reader, you might be shocked to find out that subtitlers are afforded the option to set the kerning, as well as to use non-system fonts via container embedding.

Much in the same way that the fonts embedded in a PDF can be extracted though, the fonts inside an arbitrary video container can often be extracted. Via some invocation of ffmpeg like described on StackOverflow, we can see that below.


ffmpeg -dump_attachment:t "" -i input.mkv

Visual Effects

While the usual gamut of a legible font with a discernable border to distinguish it from the surrounding graphics is commonplace, some subtitlers go the extra mile. When depicting items on a screen, it's often that the edge of the "camera" in an anime is abberated, as the light hits the sensor slightly differently.
